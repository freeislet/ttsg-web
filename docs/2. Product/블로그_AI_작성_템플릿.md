# TTSG 블로그 AI 작성 템플릿

## 개요

윈드서프 AI를 활용한 블로그 컨텐츠 작성을 위한 카테고리별 템플릿 가이드입니다. 각 템플릿은 일관된 품질과 구조를 보장하기 위해 설계되었습니다.

---

## 1. 기술 문서 템플릿 (tech)

### AI 프롬프트 템플릿

```
TTSG 블로그용 기술 문서를 작성해주세요.

**주제**: [구체적인 기술 주제]
**대상 독자**: [초급자/중급자/고급자]
**목표**: [학습 목표 또는 해결하고자 하는 문제]

다음 구조로 작성해주세요:

1. **문제 정의 및 배경**
   - 왜 이 기술이 필요한가?
   - 어떤 문제를 해결하는가?

2. **해결 방법 개요**
   - 핵심 개념 설명
   - 접근 방식 소개

3. **단계별 구현**
   - 코드 예시 포함
   - 각 단계별 상세 설명
   - 주의사항 및 팁

4. **실제 적용 사례**
   - 실무에서의 활용 방법
   - 성능 고려사항

5. **마무리**
   - 핵심 내용 요약
   - 추가 학습 자료 링크

**작성 가이드라인**:
- 코드 블록은 언어를 명시해주세요
- 복잡한 개념은 단계별로 나누어 설명해주세요
- 실무에서 바로 적용할 수 있는 실용적인 내용을 포함해주세요
- SEO를 위해 관련 키워드를 자연스럽게 포함해주세요
- 모든 내용은 한국어로 작성해주세요
```

### 메타데이터 템플릿

```yaml
---
title: "[기술명] 완전 정복: [핵심 키워드]"
description: "[기술명]을 활용한 [문제 해결/구현 방법]에 대한 실무 가이드"
pubDate: [YYYY-MM-DD]
category: "tech"
tags: ["[주요기술]", "[관련기술]", "[개발언어]", "tutorial"]
author: "TTSG"
draft: false
featured: false
heroImage: "/images/blog/tech/[이미지파일명].jpg"
---
```

### 예시 주제
- "Astro Content Collections로 블로그 만들기"
- "React 19의 새로운 기능 완전 분석"
- "Cloudflare Workers 성능 최적화 가이드"
- "TypeScript 고급 타입 활용법"

## 2. 뉴스 템플릿 (news)

### AI 프롬프트 템플릿

```
TTSG 블로그용 뉴스 기사를 작성해주세요.

**뉴스 주제**: [구체적인 뉴스 내용]
**뉴스 유형**: [업계 동향/제품 출시/기술 발표/정책 변화]
**중요도**: [높음/보통/낮음]

다음 구조로 작성해주세요:

1. **핵심 내용 요약**
   - 5W1H (누가, 언제, 어디서, 무엇을, 왜, 어떻게)
   - 핵심 포인트 3-5개

2. **상세 배경 설명**
   - 관련 배경 정보
   - 이전 상황과의 비교

3. **업계 영향 분석**
   - 개발자/기업에게 미치는 영향
   - 시장 변화 예측

4. **TTSG 관점**
   - 우리 서비스/제품과의 연관성
   - 향후 대응 방안

5. **향후 전망**
   - 예상되는 후속 조치
   - 관련 일정 및 로드맵

**작성 가이드라인**:
- 객관적이고 정확한 정보 전달에 집중해주세요
- 출처가 있는 경우 반드시 명시해주세요
- 추측성 내용은 명확히 구분해주세요
- 독자에게 실질적인 도움이 되는 인사이트를 포함해주세요
- 모든 내용은 한국어로 작성해주세요
```

## 3. 소개글 템플릿 (intro)

### AI 프롬프트 템플릿

```
TTSG 블로그용 소개글을 작성해주세요.

**소개 대상**: [제품/서비스/팀/기능]
**소개 목적**: [인지도 향상/사용법 안내/팀 소개]
**타겟 독자**: [개발자/일반 사용자/잠재 고객]

다음 구조로 작성해주세요:

1. **매력적인 시작**
   - 흥미를 끄는 도입부
   - 핵심 가치 제안

2. **개요 및 배경**
   - 무엇인지 명확한 설명
   - 왜 만들어졌는지 배경

3. **주요 특징 및 장점**
   - 핵심 기능 3-5개
   - 각 기능의 구체적인 장점
   - 경쟁 제품과의 차별점

4. **사용 방법 및 시작하기**
   - 간단한 사용법 안내
   - 첫 시작을 위한 가이드
   - 주요 사용 사례

5. **마무리 및 행동 유도**
   - 핵심 내용 요약
   - 다음 단계 안내 (CTA)
   - 문의 및 지원 정보

**작성 가이드라인**:
- 친근하고 접근하기 쉬운 톤앤매너를 사용해주세요
- 전문 용어는 쉽게 풀어서 설명해주세요
- 구체적인 예시와 사용 사례를 포함해주세요
- 독자의 행동을 유도하는 명확한 CTA를 포함해주세요
- 모든 내용은 한국어로 작성해주세요
```

## 공통 작성 가이드라인

### 품질 기준
1. **가독성**: 명확하고 이해하기 쉬운 문장
2. **정확성**: 검증된 정보와 올바른 기술 내용
3. **실용성**: 독자에게 실질적인 도움이 되는 내용
4. **일관성**: TTSG 브랜드 톤앤매너 유지

### SEO 최적화
1. **제목**: 50-60자 이내, 핵심 키워드 포함
2. **설명**: 150-160자 이내, 명확한 요약
3. **태그**: 3-5개, 관련성 높은 키워드
4. **헤딩**: H1-H6 구조적 사용
5. **키워드**: 자연스러운 키워드 배치 (2-3% 밀도)

### 이미지 가이드라인
1. **대표 이미지**: 1200x630px (소셜 미디어 최적화)
2. **본문 이미지**: 최대 800px 너비
3. **파일 형식**: WebP 우선, JPEG/PNG 대안
4. **Alt 텍스트**: 접근성을 위한 대체 텍스트 필수

### 링크 전략
1. **내부 링크**: 관련 포스트 2-3개 연결
2. **외부 링크**: 신뢰할 수 있는 출처만 사용
3. **앵커 텍스트**: 명확하고 설명적인 텍스트 사용
